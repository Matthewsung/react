1) 리액트를 사용하는 이유 사용자 인터페이스를 편하게 만들기 위해 (single page web) / 사용자 경험이 좋아진다 데이터 처리 간단(데이터와 화면의 일치가 편하다) 컴포넌트화 시켜 중복을 제거 2) webpack : 쪼개진 js / jsx 파일을 html이 실행할수 있는 js 파일로 변경해준다 3) state : 바뀔 여지가 있는 부분(상태) 4) jsx : javascript + XML //html보다 문법이 엄격하여 닫는 태그 필요하다 5) babel : createElement라는 문법을 <>를 이용한 태그로 변경해준다 6) React.Fragment : render 이후에 가장 상단에 div Tag가 필요한데 이를 쓰기 싫으면 대신 작성 / 바벨이 지원하면 <>로 대체 7) react Hooks : class로 만들던 state들을 함수로 만들수 있게 만드는것 8) node : javascript의 실행기(서버 아님) - npm init : 실행하면 package.json이 생성(react시 필요한 모든 package를 넣어줌) - npm i react react-dom : react와 react-dom 설치 - npm i -D webpack webpack-cli : react 실행시 필요한 webpack 및 webpack-cli 설치 ( -D: 개발시에만 필요 배포할때는 없어도 됨) / package.json파일의 dependencies는 배포시 사용, devDependencies는 개발시 사용 - npm i -D babel-loader @babel/core @babel/preset-env @babel/preset-react : 바벨 기본 파일, 최신문법을 옛날 문법으로 바꿔주는 파일, jsx같은 파일을 알아볼수있는 파일, babel과 webpack을 연결하는 파일 설치 - client.jsx 나 개인이 만든 jsx파일을 app.js파일로 하나로 만들어(webpack 사용) html파일에서는 app.js 파일만 로드한다 - loader의 preset은 plugin들의 집합으로 target으로 각각 지정도 가능 - module이나 rules에서 지원하는 plugin외에 적용하는 plugin을 지원
